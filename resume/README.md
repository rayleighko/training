뿐만 아니라 작업물의 완성을 목표로 진행했기 때문에 소스코드의 질이 떨어졌습니다. 이에 리펙토링의 필요성을 느꼈고, 프로젝트를 하나하나 수정하는 것보다 새로운 프로젝트를 만드는 것이 나을 것 같다고 생각해 새로운 프로젝트(Boggle)를 만들게 되었습니다.
